Gestione Richieste (Sviluppo Software)

La funzionalità è accessibile dal cavaliere "Gestione Richieste" agli utenti abilitati (Ditta Informatica, ORSA e Regione) e consente di gestire il flusso delle attività di sviluppo.



Tipologia di moduli


Una richiesta può essere composta dai seguenti tipi di modulo:
  • Modulo A: Richieste Regionali
  • Modulo B: Requisiti
  • Modulo C: Proposte implementazioni
  • Modulo CH: Chiarimenti
  • Modulo D: Affidamenti
  • Altri allegati generici

A seguito di una nuova richiesta, la Regione allega un modulo A iniziale che viene poi formalizzato in un modulo B, la Ditta Informatica produce a questo punto una o più revisioni del modulo C (ed eventuali moduli CH) da sottoporre al referente che li valida tramite il modulo D dopo il quale vengono avviati gli sviluppi.

Stati


Una richiesta può trovarsi nei seguenti stati:

StatoSpiegazioneChi può settarlo
APERTO Stato di default di una richiesta appena inserita. Regione (automaticamente inserendo la richiesta)
Ditta Informatica
STANDBY La richiesta è stata messa in pausa. Ditta Informatica
CONSEGNATO Sviluppi e test interni sono terminati e la funzionalità è stata messa a disposizione per il collaudo esterno. Ditta Informatica
COLLAUDATO Il collaudo esterno è terminato con esito positivo. Referente della richiesta
ANNULLATO La richiesta è stata annullata. Ditta Informatica
CHIARIMENTI IN CORSO Sono in corso dei chiarimenti tra la ditta e il referente. Ditta Informatica

Il possibile flusso degli stati è:
  • APERTO -> CONSEGNATO (Possibile solo se presenti i moduli C e D)
  • APERTO -> STANDBY
  • APERTO -> ANNULLATO
  • CONSEGNATO -> COLLAUDATO (Possibile solo se presenti i moduli C e D)
  • STANDBY -> APERTO
  • ANNULLATO -> APERTO

Gestione dei referenti


In fase di inserimento di una nuova richiesta andrà indicato il referente, ossia la persona con cui la Ditta Informatica si interfaccerà come responsabile.
La lista di referenti (selezionabili tra il personale regionale e ORSA/OEV) potrà essere modificata on-demand.



Il referente potrà essere modificato (da ORSA/OEV o Regione) nella pagina di dettaglio della richiesta finchè essa non si trova nello stato CONSEGNATO o COLLAUDATO.



Settaggio della priorità


Il Responsabile Regione potrà definire la priorità della richiesta (che di default sarà NON DEFINITA).
La priorità potrà essere modificata nella pagina di dettaglio della richiesta finchè essa non si trova nello stato CONSEGNATO o COLLAUDATO.



Note


Nel dettaglio di una richiesta sarà possibile aggiungere delle note.
Le note saranno aggiunte in coda alle altre eventualmente presenti e riporteranno l'utente che le ha inserite.


Inserimento di una nuova richiesta




L'inserimento di una nuova richiesta è a carico della Regione ed avviene tramite il caricamento di un Modulo A, mediante il bottone giallo "Aggiungi modulo A".



In fase di aggiunta occorre indicare:
  • Descrizione della richiesta
  • Tags della richiesta (Insieme di termini riguardanti l'argomento trattato, per favorire la ricerca)
  • File (vedi paragrafo "Sintassi del nome file")
  • Referente

Sintassi del nome file


Il nome del file da allegare deve rispettare la seguente sintassi:
Modulo [Lettera identificativa del modulo] [Progressivo della richiesta]

Es.

ModuloA045.pdf
Modulo_A 111.odt
Modulo A_006.doc
Modulo-A-900.pdf
Dal nome del file saranno estratti il tipo di modulo ed il progressivo della richiesta.

Una volta generata la richiesta, essa apparirà in una riga dedicata della tabella.


Da qui sarà possibile aggiungere i moduli B, C, CH, D tramite il link "Aggiungi" in corrispondenza della rispettiva colonna. In questo caso non sarà richiesta alcuna sintassi particolare nel nome del file.
Sarà inoltre possibile applicare dei filtri per facilitare la ricerca.
